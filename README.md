# Drebolbot

Telegram-бот для внутренней работы с продажами, расчётами, FunPay, AI-инструментами, тикетами и служебным администрированием.

Репозиторий: `https://github.com/pratokwau/drebol-bot.git`

## Что умеет бот

- вести журнал прибыли и считать статистику по FunPay и PlayerOK;
- выполнять расчёты для FunPay и PlayerOK;
- открывать AI-чат с поддержкой текста, голосовых и фото;
- озвучивать ответы AI через `edge-tts`;
- работать с FunPay продажами и автоподбором закупа;
- хранить минимальные цены, коэффициенты СБП и данные по демпингу;
- показывать статус работы бота и фиксировать даунтаймы;
- принимать тикеты от пользователей и давать администратору интерфейс ответа;
- управлять доступами, группами, рассылками и пользовательскими настройками;
- обновляться из Git через админ-панель.

## Основные команды

### Для пользователей

- `/start` - главное меню
- `/about` - информация о боте
- `/settings` - персональные настройки уведомлений и голоса AI
- `/help` - тикет администратору
- `/cancel` - выход из текущего сценария
- `/rassstart` - расчёт прибыли для FunPay
- `/playerokrass` - расчёт прибыли для PlayerOK
- `/saveprofit` - журнал прибыли, статистика, ручные записи
- `/ai` - AI-ассистент

### Для администратора

- `/admin` - админ-панель
- `/funpayauto` - меню FunPay Auto
- `/funpayau` - алиас команды FunPay Auto
- `/minprice` - управление минимальными ценами
- `/aisettings` - настройка AI-ключей

## Модули проекта

### Администрирование

- выдача и отзыв доступа;
- список пользователей и карточки пользователей;
- заметки по пользователям;
- блокировка отдельных команд для конкретных пользователей;
- обработка заявок на доступ;
- рассылка и отправка личных сообщений;
- управление разрешёнными группами;
- редактирование текста `/about`;
- запуск обновления из Git и перезапуск сервиса.

### FunPay

- хранение `golden_key` и `user-agent`;
- просмотр продаж с пагинацией;
- поиск заказа по номеру или ссылке;
- ручной и автоматический ввод закупа;
- пересчёт прибыли по заказам;
- ручное исправление цены продажи, если API вернул её некорректно;
- автоподбор закупа по `minprice` и сертификатам.

### MinPrice и демпинг

- игры и товары с хранением закупа, минимальной цены и кэшбека;
- привязка одного или нескольких `offer_id` к товару;
- расчёт минимальной цены с учётом комиссии;
- поиск и проверка коэффициентов СБП;
- импорт игр из FunPay;
- проверка актуальности данных;
- отдельные модули демпинга и сертификатов.

### AI

- провайдеры: Groq и OpenRouter;
- fallback между несколькими моделями;
- отдельные чаты для пользователя;
- обработка текста, голосовых и фото;
- распознавание голосовых через Groq Whisper;
- озвучка ответов через `edge-tts`;
- хранение истории по пользователям в `users/<id>/ai_chats.json`.

### Учёт прибыли

- ручное добавление сделок по FunPay и PlayerOK;
- журнал записей с пагинацией;
- редактирование и удаление записей;
- статистика за день, неделю, месяц и свой период;
- ежедневные уведомления с отчётом;
- отдельный админский отчёт с изменениями коэффициентов СБП.

### Тикеты и статус

- пользовательский чат с администратором через `/help`;
- хранение истории тикетов в JSON;
- просмотр медиа и истории тикета администратором;
- heartbeat-файл и журнал сбоев;
- уведомления о перезапуске бота.

## Быстрая установка на сервер

Проект содержит готовый установщик для Linux-сервера с `systemd`.

```bash
bash <(curl -Ls https://raw.githubusercontent.com/pratokwau/drebol-bot/main/install.sh)
```

Или:

```bash
wget https://raw.githubusercontent.com/pratokwau/drebol-bot/main/install.sh -O install.sh && bash install.sh
```

Что делает установщик:

1. Проверяет запуск от `root`.
2. Устанавливает `git`, `python3`, `python3-pip` и пакет `pythonX.Y-venv`.
3. Полностью пересоздаёт установку в `/root/drebolbot`.
4. Клонирует репозиторий.
5. Запускает первичную настройку через `install.py`.
6. Создаёт `.venv` и ставит зависимости.
7. Регистрирует `systemd`-сервис `drebolbot`.
8. Включает автозапуск и запускает сервис.

Важно: установщик удаляет старую папку `/root/drebolbot` перед переустановкой.

## Ручной запуск

### 1. Клонирование

```bash
git clone https://github.com/pratokwau/drebol-bot.git
cd drebol-bot
```

### 2. Виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 4. Создание `.env`

Минимальный пример:

```env
TOKEN=your_telegram_bot_token
ADMIN_ID=123456789
AUTH_FILE=authorized.json
FP_TOKEN=
DATA_DIR=data
INVENTORY_FILE=data/inventory.json
GROQ_API_KEY=
OPENROUTER_API_KEY=
```

### 5. Подготовка файлов

```bash
mkdir -p data
mkdir -p users
printf '[]\n' > authorized.json
printf '{}\n' > data/inventory.json
```

### 6. Запуск

```bash
python main.py
```

## Переменные окружения

Используются значения из `config.py`:

- `TOKEN` - токен Telegram-бота;
- `ADMIN_ID` - Telegram ID главного администратора;
- `AUTH_FILE` - путь к JSON со списком авторизованных пользователей;
- `FP_TOKEN` - дополнительная переменная для FunPay-логики;
- `DATA_DIR` - базовая папка данных;
- `INVENTORY_FILE` - файл инвентаря;
- `GROQ_API_KEY` - ключ Groq;
- `OPENROUTER_API_KEY` - ключ OpenRouter.

На практике:

- `TOKEN` и `ADMIN_ID` обязательны для запуска;
- `GROQ_API_KEY` и `OPENROUTER_API_KEY` можно заполнить позже через админ-интерфейс AI;
- для FunPay Auto администратору дополнительно нужно задать `golden_key` и `user-agent` внутри бота.

## Как хранится доступ

- `authorized.json` содержит список ID пользователей с полным доступом;
- админ `ADMIN_ID` автоматически добавляется в авторизованные при старте, даже если файл пустой;
- группы хранятся в `data/authorized_groups.json`;
- запросы доступа хранятся в `data/access_requests.json`.

## Обновление проекта

В админ-панели бот умеет:

- проверить, есть ли новая версия в удалённой ветке;
- выполнить `git fetch`;
- выполнить жёсткое выравнивание к `origin/<branch>`;
- перезапустить сервис `drebolbot`.

Важно:

- логика обновления рассчитана на установленный серверный экземпляр;
- пользовательские данные в `data/`, `users/` и `.env` самим кодом обновления не переписываются;
- механизм обновления использует `git reset --hard` внутри `update_manager.py`, поэтому его стоит применять только на серверной установке, где репозиторий используется как рабочая копия приложения.

## Планировщик задач

В `main.py` зарегистрированы фоновые задачи:

- напоминания о незаполненных заказах;
- ежеминутная проверка времени ежедневных отчётов;
- ежеминутная рассылка персональных daily report;
- heartbeat каждые 60 секунд;
- проверка и логирование даунтайма после старта.

Часовой пояс планировщика: `Europe/Moscow`.

## Структура хранения данных

### Корневые файлы

- `main.py` - точка входа
- `loader.py` - `Bot`, `Dispatcher`, авторизация
- `config.py` - загрузка переменных окружения
- `database.py` - служебная база проекта
- `update_manager.py` - проверка обновлений и перезапуск
- `install.py` - первичная настройка
- `install.sh` - серверный установщик

### Каталоги

- `handlers/` - вся прикладная логика бота
- `states/` - FSM состояния
- `middlewares/` - middleware ограничения команд
- `FunPayAPI/` - встроенная библиотека для работы с FunPay
- `data/` - общие JSON-файлы проекта
- `users/` - пользовательские данные, AI-чаты, minprice и прочее
- `base/` - пользовательские SQLite-базы настроек и локальных данных

### Часто используемые файлы данных

- `authorized.json` - авторизованные пользователи
- `data/authorized_groups.json` - разрешённые группы
- `data/access_requests.json` - заявки на доступ
- `data/command_restrictions.json` - точечные блокировки команд
- `data/tickets.json` - история тикетов
- `data/about.json` - текст команды `/about`
- `data/mute_restart.json` - мут уведомлений о рестарте
- `data/sbp_changes_today.json` - изменения коэффициентов СБП за день
- `data/restart_notice.json` - служебное сообщение после перезапуска

## Особенности реализации

- настройки пользователей хранятся в SQLite по пользователям, при этом есть миграция из старого `data/user_settings.json`;
- AI-чаты и часть справочных данных хранятся в JSON по пользователям;
- бот работает через long polling, не через webhook;
- сервис устойчив к падению polling-цикла и пытается перезапуститься сам через 5 секунд;
- после рестарта бот может уведомлять авторизованных пользователей, если у них включена эта опция.

## Ограничения и нюансы

- серверный установщик ориентирован на путь `/root/drebolbot` и Linux с `systemd`;
- локально на macOS или Windows лучше использовать ручной запуск;
- некоторые файлы в репозитории являются вспомогательными или заготовками, но в `main.py` подключены не все из них;
- функциональность FunPay и AI зависит от внешних ключей, cookies и доступности внешних API.

## Полезно при эксплуатации

Если бот уже установлен как сервис:

```bash
systemctl status drebolbot
journalctl -u drebolbot -f
```

Если нужен первый запуск без установщика, достаточно подготовить `.env`, `authorized.json`, `data/` и запустить `python main.py`.
